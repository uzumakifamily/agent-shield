How it works

What happens every time your agent takes an action

AgentShield sits in the middle. Your agent proposes — AgentShield decides. Nothing risky executes without going through the check.

🤖

an action

Your AI agent decides to do something — delete a file, send an email, run a shell command, deploy to production.

🛡️

→

✓
ALLOW
Low risk. Action runs immediately. Logged to the immutable audit trail. No human needed.
⏸
PAUSE
Risk threshold or rule match. Action is held. You get a Telegram alert. Approve or deny from dashboard or phone.
✕
BLOCK
High risk or explicit deny rule. Action is stopped immediately. Agent is notified. Nothing happens until you change the rule.

Wrap any action — production_deploy example
const shield = require('agent-shield-core');

// Call this BEFORE your agent executes any action
const result = await shield.run(
  {
    workspaceId: 'YOUR_WORKSPACE_ID',  // from Settings
    projectId:   'my-project',
    agent:       'my-agent',
    actionType:  'production_deploy',
    source:      'api',
    payload:     { service: 'api', env: 'prod' }
  },
  async () => {
    // Your real action — only runs if verdict is ALLOW
    return await deployToProduction();
  }
);

// result.verdict is 'ALLOW', 'PAUSE', or 'BLOCK'
if (!result.proceed) {
  console.log('Blocked or paused:', result.verdict, result.reason);
  // result.approval_id is set if PAUSE — poll or wait for webhook
}

No Python SDK needed — use the REST API directly with requests. Add this check before any action your agent takes.

Helper function + file_delete example
import requests

SHIELD_API = "https://agentshield.dev"
API_KEY    = "YOUR_API_KEY"

def check_action(action_type, payload, agent="my-agent"):
    """Call AgentShield before any risky action."""
    r = requests.post(
        f"{SHIELD_API}/api/actions",
        headers={
            "Authorization": f"Bearer {API_KEY}",
            "Content-Type":  "application/json"
        },
        json={
            "action_type": action_type,
            "payload":     payload,
            "context":     {"agent": agent, "source": "api"}
        }
    )
    return r.json()

# --- Example: check before deleting a file ---
result = check_action("file_delete", {"path": "/data/users.csv"})

if result["verdict"] == "ALLOW":
    delete_file("/data/users.csv")        # safe to run
elif result["verdict"] == "PAUSE":
    print("Waiting for approval:", result.get("approval_id"))
elif result["verdict"] == "BLOCK":
    print("Action blocked:", result["reason"])

https://agentshield.dev/webhooks/YOUR_WORKSPACE_ID
POST body — send_email example
{
  "action_type": "send_email",
  "payload": {
    "to":      "client@example.com",
    "subject": "Your weekly report is ready",
    "body":    "Here is the summary..."
  },
  "context": {
    "agent":  "zapier-workflow",
    "source": "webhook"
  }
}
In Zapier / Make / n8n: set method to POST, Content-Type to application/json, paste the body above. The URL itself authenticates the request — no Authorization header is needed. The verdict (ALLOW / PAUSE / BLOCK) is returned in the response body.
